_01_ Memory & VRAM Protection Layer

The Problem

Processing high-resolution bitmap assets (e.g., raw photos exceeding 50MB) inside an unmanaged web context causes instant VRAM/RAM starvation. This triggers a destructive browser tab crash (Out of Memory) when trying to commit massive arrays into hardware-accelerated canvas contexts.

The Architecture

PixelForge enforces a strict, isolated two-tier defensive validation pipeline before allocating heavy system buffers:

  • MAX_FILE_SIZE = 20MB hard limit.
  • MAX_WIDTH / MAX_HEIGHT = 4096px boundaries.

_02_ Multi-Threading & Race Elimination

The Problem

Color quantization and MaxRects bin packing are heavy operations offloaded to Web Workers. Rapidly triggering actions or changing parameters mid-calculation causes overlapping worker events. An outdated thread could finish after a newer one, corrupting the canvas context with stale data.

The Architecture

Resolved by implementing a strict Thread Hot-Restart & Instant Termination pattern via the native Web Worker .terminate() controller API. 

function initImageWorker() {
  if (imageWorker) imageWorker.terminate(); // Annihilate active ghost thread
  
  imageWorker = new Worker(
    new URL("./core/workers/image.worker.ts", import.meta.url),
    { type: "module" }
  );
}

_03_ Persistent Cyclic DB History Buffer

The Problem

To support undo/redo trees, transactions are written to IndexedDB. Historically, offset subtraction caused obsolete states at index 0, 1, 2 to remain pinned on disk permanently, causing heavy disk bloat during long sprite sessions.

The Architecture

The cleaning routine operates as a creeping sequential garbage collector that aggressively purges the absolute historical tail up to the safe sliding window limit: 

const oldestAllowedIndex = this.maxIndex - this.MAX_STATES;
if (oldestAllowedIndex >= 0) {
  for (let i = 0; i <= oldestAllowedIndex; i++) {
    await db.delete("history", i);
  }
}

_04_ Binary Alpha Normalization (QOI Guard)

The Problem

Graphics software often leaves invisible garbage RGB data inside fully transparent pixels (Alpha == 0). The QOI encoder wasted bytes calculating color shifts for invisible pixels, causing file size bloat and dirty edge noise on microcontrollers.

The Architecture

PixelForge injects a mutative Alpha Cleansing Pass. If a pixel's alpha registers as transparent, its RGB metadata is forcefully zeroed out at the byte level: 

if (a === 0) {
    r = 0; g = 0; b = 0;
}

_05_ Hardware Extensions Specification

1-bit Monochrome Export: Designed for low-power OLED matrices (SSD1306). Converts standard 32-bit arrays into compact streams using a custom luminance threshold algorithm, packing 8 distinct pixels into a single physical byte (MSB-first format).

Garmin Look-Up Table (LUT): Maps stitched atlases straight onto the native 16-color Garmin hardware palette, exporting index byte arrays accompanied by exact byte frame offsets (garmin_sprite_offsets) for O(1) fast execution on wearables.
